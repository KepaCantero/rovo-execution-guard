# [RB-044] Prettier Docs - Code Formatting, Consistent Style

> Fuente: Prettier Docs - Code formatting, consistent style

## Reglas

### GIT-CI-044-01
**DEFINICION:** La configuracion de Prettier debe estar centralizada en un unico archivo `.prettierrc` en la raiz del proyecto con: `printWidth: 100`, `singleQuote: true`, `trailingComma: 'all'`, `semi: true`, `tabWidth: 2`, `arrowParens: 'always'`.
**VALOR:** La configuracion unica elimina disputas de estilo, reduce el ruido en diffs de git, y asegura que todo el codigo formateado por cualquier desarrollador o herramienta produce el mismo resultado.
**IMPLEMENTACION:** Crear `.prettierrc` con los valores especificados. Anadir `.prettierignore` con `node_modules/, dist/, coverage/, *.min.js, src/types/api-generated.ts`. Ejecutar `npx prettier --write "src/**/*.ts"` como script en `package.json`: `"format": "prettier --write 'src/**/*.ts' 'tests/**/*.ts'"`.
**AUDITORIA:** Ralph verifica que `.prettierrc` existe en la raiz con los parametros especificados, que `.prettierignore` excluye archivos generados, y que el script `format` esta en `package.json`.

### GIT-CI-044-02
**DEFINICION:** Prettier debe ejecutarse como pre-commit hook via lint-staged, formateando solo los archivos staged; nunca ejecutar Prettier manualmente sobre todo el proyecto como parte del flujo normal.
**VALOR:** El pre-commit hook formatea incrementalmente solo lo que cambia sin penalizar el tiempo de commit; una ejecucion full solo se necesita en la configuracion inicial.
**IMPLEMENTACION:** En `package.json`: `"lint-staged": { "*.{ts,tsx}": ["prettier --write", "eslint --fix"] }`. Configurar Husky: `npx husky add .husky/pre-commit 'npx lint-staged'`. Esto formatea y lint solo los archivos modificados antes de cada commit.
**AUDITORIA:** Ralph verifica que el hook pre-commit de Husky ejecuta lint-staged y que lint-staged incluye Prettier para archivos TypeScript.

### GIT-CI-044-03
**DEFINICION:** El pipeline CI debe incluir un step `prettier --check` que falle si algun archivo no esta formateado; este check es no-negociable y no se puede deshabilitar con `--no-verify`.
**VALOR:** El check en CI es la ultima linea de defensa contra codigo mal formateado que paso el hook local (ej. commits con `--no-verify` o edits en GitHub web).
**IMPLEMENTACION:** Anadir step en GitHub Actions: `- name: Check Prettier\n  run: npx prettier --check 'src/**/*.ts' 'tests/**/*.ts'`. Este step debe ejecutarse antes de los tests y fallar el pipeline inmediatamente si detecta diferencias de formato.
**AUDITORIA:** Ralph verifica que el workflow CI contiene el step `prettier --check` y que falla el pipeline cuando detecta archivos no formateados.

### GIT-CI-044-04
**DEFINICION:** Las reglas de ESLint que conflictuan con Prettier (indentacion, comillas, punto y coma, trailing commas) deben deshabilitarse usando `eslint-config-prettier`; las decisiones de formato son dominio exclusivo de Prettier.
**VALOR:** Evita la friccion entre ESLint y Prettier donde uno formatea y el otro desformatea en un loop infinito, causando fallas de CI intermitentes.
**IMPLEMENTACION:** Instalar `eslint-config-prettier` y anadir `"prettier"` como ultimo elemento en el array `extends` de `.eslintrc`: `extends: ['eslint:recommended', '@typescript-eslint/recommended', 'prettier']`. Esto deshabilita todas las reglas de ESLint que conflictuan con Prettier.
**AUDITORIA:** Ralph verifica que `.eslintrc` incluye `prettier` en el array `extends` como ultimo elemento y que no existen reglas de formato duplicadas entre ESLint y Prettier.

### GIT-CI-044-05
**DEFINICION:** Los archivos generados automaticamente (`src/types/api-generated.ts`, `coverage/`, `dist/`) deben excluirse del formateo en `.prettierignore` y marcarse con un header de autogeneracion para evitar ediciones manuales accidentales.
**VALOR:** Formatear archivos generados es desperdicio de recursos y puede causar conflicts en git; peor aun, ediciones manuales en archivos generados se pierden en la proxima generacion.
**IMPLEMENTACION:** Crear `.prettierignore` con entradas: `src/types/api-generated.ts`, `coverage/`, `dist/`, `node_modules/`. En scripts de generacion, inyectar header: `/* eslint-disable prettier/prettier */\n// @generated by openapi-typescript. DO NOT EDIT.`.
**AUDITORIA:** Ralph verifica que `.prettierignore` contiene las rutas de archivos generados y que estos archivos llevan un header que indica su naturaleza autogenerada.
